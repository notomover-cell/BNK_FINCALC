# BNK 금융계산기 — 프로젝트 기획서

> **목적:** 부산은행 창구 직원이 네오비스(행내 시스템) 위에 항상 띄워놓고 사용하는 윈도우 데스크톱 금융계산기
> **핵심 페인포인트:** 고객 응대 중 만기예상금, DSR, 환전 계산 시 기존 화면에서 이탈해야 하거나 엑셀 임시방편 사용
> **환경:** 은행 내부 폐쇄망 (인터넷 불가), Windows PC (저사양 고려, Edge Chromium 설치됨)
> **배포:** PyInstaller 단일 exe (Python 포함) → 공유폴더 또는 메신저로 배포

---

## 1. 기술 스택

| 항목 | 선택 | 비고 |
|------|------|------|
| 프론트엔드 | HTML / CSS (Tailwind) / JS | Google Stitch로 디자인 후 export |
| 데스크톱 래핑 | Python + pywebview (EdgeChromium 렌더러) | 메모리 40~60MB, Always on Top 지원 |
| 로컬 DB | SQLite | 계산 이력 저장 |
| 설정 파일 | JSON | LTV/DTI 규정, 환율 등 외부 설정 |
| 빌드/배포 | PyInstaller | 단일 exe 패키징 → 행내 배포 |

## 2. 앱 구조 및 윈도우 기능

### 2.1 플로팅 창
- **Always on Top:** 네오비스 위에 항상 표시
- **트레이 아이콘:** 최소화 시 시스템 트레이로 내려감, 단축키(예: Ctrl+Shift+C)로 호출
- **플로팅 버튼:** 화면 모서리에 작은 버튼 상시 표시, 클릭 시 메인 창 열림
- **크기 조절:** 드래그로 창 크기 변경 가능, 최소 크기 제한(400×600px)

### 2.2 탭 네비게이션
상단 탭 바: `예금` | `적금` | `대출` | `주담대 종합` | `LTV` | `DTI` | `DSR` | `중도상환` | `환전/송금` | `이력`

### 2.3 공통 UI 요소
- 모든 금액 입력: 천 단위 콤마 자동 포맷, 한글 단위 표시 (예: "1,000만 원")
- 빠른 금액 버튼: `+10만` `+100만` `+1,000만` `+1억`
- 빠른 기간 버튼: `+1년` `+5년` `+10년`
- 초기화 / 계산하기 버튼
- 계산 결과 영역: 접기/펼치기 가능
- **원단위 처리:** 모든 금액 계산 결과는 원 미만 절상 (올림) 처리

### 2.4 메모리 기능
- **계산 결과 임시저장:** 각 계산 결과에 `[M+]` 버튼 → 클립보드 영역에 저장
- **다른 계산에서 불러오기:** 금액 입력 필드 옆 `[MR]` 버튼 → 저장된 값 목록에서 선택 삽입
- **메모리 슬롯:** 최대 10개, 라벨 자동 생성 (예: "대출 계산 - 월 상환액 837,855원")

### 2.6 멀티 윈도우
- 계산기를 여러 개 동시에 띄울 수 있도록 지원 (고객 동시 상담 대응)
- 트레이 아이콘 메뉴 또는 단축키로 새 창 열기

### 2.7 계산 이력 자동 저장
- 계산 실행 시마다 SQLite에 자동 기록
- 저장 항목: 계산 종류, 입력값, 결과값, 타임스탬프
- 고객 식별정보 없음 (개인정보 이슈 방지)
- 직원이 메모를 직접 추가/수정 가능
- 이력 탭에서 조회: 날짜별 필터, 계산 종류별 필터, 검색

### 3.1 예금 계산기

**설명:** 일정 금액을 한번에 납입하는 정기예금의 만기 수령액 계산

**입력 필드:**

| 필드명 | 타입 | 옵션 | 기본값 |
|--------|------|------|--------|
| 예치금액 | 금액 입력 | 빠른 금액 버튼 | 0 |
| 예금기간 | 숫자 입력 | 년 / 월 토글 | 년 |
| 연이자율 | 소수점 입력 | — | 0% |
| 이자 방식 | 토글 | 단리 / 월복리 | 단리 |
| 이자과세 | 토글 | 일반과세(15.4%) / 비과세(0%) / 세금우대(9.5%) / 저율과세(1.4%) | 일반과세 |

**계산 공식:**

```
[단리]
세전이자 = 원금 × 연이율 × (기간월수 / 12)

[월복리]
세전이자 = 원금 × (1 + 연이율/12)^기간월수 - 원금

[세금]
일반과세: 세전이자 × 15.4%
비과세: 0
세금우대종합저축: 세전이자 × 9.5%
저율과세(상호금융): 세전이자 × 1.4%  (농특세만)

[결과]
세후 수령액 = 원금 + 세전이자 - 세금
```

**결과 표시:**
- 예치금액 (원금)
- 세전이자
- 세금 (빨간색)
- **총 수령액** (강조 색상)

### 3.2 적금 계산기

**설명:** 매월 일정 금액을 납입하는 정기적금의 만기 수령액 계산

**입력 필드:**

| 필드명 | 타입 | 옵션 | 기본값 |
|--------|------|------|--------|
| 월납입액 | 금액 입력 | 빠른 금액 버튼 | 0 |
| 적립기간 | 숫자 입력 | 년 / 월 토글 | 년 |
| 연이자율 | 소수점 입력 | — | 0% |
| 이자 방식 | 토글 | 단리 / 월복리 | 단리 |
| 이자과세 | 토글 | 일반과세(15.4%) / 비과세(0%) / 세금우대(9.5%) / 저율과세(1.4%) | 일반과세 |

**계산 공식:**

```
[단리]
총 납입액 = 월납입액 × 기간월수
세전이자 = 월납입액 × 연이율/12 × 기간월수 × (기간월수 + 1) / 2

[월복리]
월이율 = 연이율 / 12
세전이자 = 월납입액 × ((1 + 월이율)^기간월수 - 1) / 월이율 - 총납입액

[세금] — 예금과 동일

[결과]
총 수령액 = 총 납입액 + 세전이자 - 세금
```

**결과 표시:**
- 총 납입액
- 세전이자
- 세금
- **총 수령액** (강조)

---

### 3.3 대출 계산기

**설명:** 대출 시 월 상환액, 총 이자, 총 상환금액 계산

**입력 필드:**

| 필드명 | 타입 | 옵션 | 기본값 |
|--------|------|------|--------|
| 대출금액 | 금액 입력 | 빠른 금액 버튼 | 0 |
| 대출기간 | 숫자 입력 | 년 / 월 토글 | 년 |
| 대출금리 | 소수점 입력 | — | 0% |
| 상환방법 | 토글 | 원리금균등 / 원금균등 / 만기일시 / 체증식 | 원리금균등 |

**계산 공식:**

```
월이율 = 연이율 / 12
n = 기간월수

[원리금균등상환]
월 상환액 = 대출금액 × 월이율 × (1 + 월이율)^n / ((1 + 월이율)^n - 1)
총 상환액 = 월 상환액 × n
총 이자 = 총 상환액 - 대출금액

[원금균등상환]
월 원금 = 대출금액 / n
k번째 월 이자 = (대출금액 - 월 원금 × (k-1)) × 월이율
k번째 월 상환액 = 월 원금 + k번째 월 이자
총 이자 = Σ(각 월 이자)
→ 첫 달 상환액과 마지막 달 상환액이 다름

[만기일시상환]
월 이자 = 대출금액 × 월이율
총 이자 = 월 이자 × n
만기 시 원금 + 마지막 월 이자 일시 상환

[체증식분할상환]
체증률 g = 원리금균등 월 상환액 기준 역산 (초기 상환액이 낮고 매년 일정 비율 증가)
1차년도 월 상환액 = 대출금액 × 월이율 × (1+g)^0 × 보정계수
k차년도 월 상환액 = 1차년도 × (1+g)^(k-1)
→ 보금자리론 등 정책대출에서 사용, 초기 상환 부담 경감
```

**결과 표시:**
- 대출원금
- 총 대출이자
- 총 상환금액
- **매월 상환금액** (강조, 원금균등은 "첫 달 / 마지막 달" 범위 표시)
- `[월별 상환 금액 더보기]` 버튼 → 월별 상환 스케줄 테이블 (회차, 원금, 이자, 잔액)

### 3.4 LTV 계산기 (담보인정비율)

**설명:** 주택 담보 대출 시 LTV 기준 최대 대출 가능 금액 산출

**입력 필드:**

| 필드명 | 타입 | 옵션 | 기본값 |
|--------|------|------|--------|
| 대출목적 | 토글 | 신규주택구입 / 생활안정자금 | 신규주택구입 |
| 보유 주택 수 | 토글 | 생애최초 / 서민실수요 / 무주택 / 1주택(처분조건부) / 2주택이상 | 무주택 |
| 규제지역 여부 | 토글 | 규제지역(수도권) / 비규제지역 | 비규제지역 |
| 담보대상 주택 시세 | 금액 입력 | 빠른 금액 버튼 | 0 |
| 보유 대출 잔액 | 금액 입력 | (생활안정자금 선택 시 표시) | 0 |
| 임차보증금 | 금액 입력 | (생활안정자금 선택 시 표시) | 0 |
| 소액임차보증금 최우선변제금액 | 금액 입력 | 정책변수 기본값 로드, 직원 수정 가능 | 정책변수 기본값 |

**LTV 비율 규정 — 정책변수 JSON (`ltv_rules.json`)**

```json
{
  "last_updated": "2026-04-05",
  "신규주택구입": {
    "규제지역": {
      "생애최초": 70,
      "서민실수요": 60,
      "무주택": 40,
      "1주택_처분조건부": 50,
      "2주택이상": 0
    },
    "비규제지역": {
      "생애최초": 80,
      "무주택": 70,
      "1주택": 60,
      "2주택이상": 50
    }
  },
  "생활안정자금": {
    "규제지역": {
      "1주택": 40,
      "2주택이상": 0
    },
    "비규제지역": {
      "1주택": 70,
      "2주택이상": 60
    }
  },
  "절대한도_수도권_규제지역": {
    "15억이하": 60000,
    "15억초과_25억이하": 40000,
    "25억초과": 20000,
    "생활안정자금_1주택": 10000,
    "생활안정자금_2주택이상": 0
  },
  "소액임차보증금": {
    "서울": 5500,
    "수도권_과밀억제권역": 4800,
    "광역시_세종_안산_용인_김포_광주": 2800,
    "기타": 2000
  }
}
```

> ⚠️ 이 규정은 수시로 변경됩니다(6.27/9.7/10.15 대책 등). JSON 파일만 수정하면 앱 재빌드 없이 반영됩니다.
> ⚠️ LTV 비율은 정책변수이며, 직원이 현장 규정에 맞게 설정 화면에서 수정 가능합니다.

**계산 공식:**

```
[신규주택구입]
LTV 한도 = 주택 시세 × LTV 비율(%) - 보유 대출 잔액 - 소액임차보증금

[생활안정자금]
LTV 한도 = 주택 시세 × LTV 비율(%) - 보유 대출 잔액 - 임차보증금 - 소액임차보증금

[절대한도 적용 (규제지역)]
최대 대출 가능액 = min(LTV 한도, 절대한도)
```

**결과 표시:**
- 적용 LTV 비율 (예: "40%")
- LTV 기준 한도
- **절대한도 적용 여부** (해당 시 "절대한도 6억원 적용" 표시)
- **최대 대출 가능 금액** (강조)
- 조건 요약 (예: "신규주택구입, 무주택, 규제지역")
- ⚠️ "입력한 정보 기반 예상 금액이므로 정확한 금액은 금융사에서 확인하시기 바랍니다."
- ⚠️ 규정 기준일 표시 (`last_updated` 기반) + 30일 이상 미갱신 시 경고 배지 표시

### 3.5 DTI 계산기 (총부채상환비율)

**설명:** 주택담보대출 한도 설정을 위한 DTI 비율 산출

**입력 필드:**

| 필드명 | 타입 | 옵션 | 기본값 |
|--------|------|------|--------|
| 연소득 | 금액 입력 | 빠른 금액 버튼 | 0 |
| 주택 담보 대출 금액 | 금액 입력 | 빠른 금액 버튼 | 0 |
| 주택 담보 대출 기간 | 숫자 입력 | 년 / 월 토글 | 년 |
| 주택 담보 대출 금리 | 소수점 입력 | — | 0% |
| 주택 담보 대출 상환방법 | 토글 | 원리금균등 / 원금균등 / 만기일시 | 원리금균등 |
| 보유 주담대 연원리금 상환액 | 금액 입력 | (기존 주담대 원금+이자) | 0 |
| 기타대출 연이자 상환액 | 금액 입력 | (신용대출 등 이자만) | 0 |

**계산 공식 (신DTI 기준):**

```
신규 주담대 연원리금 = 대출 상환방법에 따른 연간 원리금 (3.3 대출 계산기 공식 활용)
DTI = (신규 주담대 연원리금 + 보유 주담대 연원리금 + 기타대출 연이��) / 연소득 × 100
```

> 📌 신DTI: 보유 주담대는 원리금(원금+이자) 포함, 기타 대출(신용대출 등)은 이자만 포함

**결과 표시:**
- 연원리금 상환액
- **DTI 비율** (강조, 예: "17.19%")
- 조건 요약

### 3.6 DSR 계산기 (총부채원리금상환비율)

**설명:** 모든 대출의 연간 원리금 상환액을 기반으로 DSR 비율 산출 (스트레스 DSR 포함)

**입력 필드:**

| 필드명 | 타입 | 옵션 | 기본값 |
|--------|------|------|--------|
| 연소득 | 금액 입력 | 빠른 금액 버튼 | 0 |
| 주택 담보 대출 금액 | 금액 입력 | — | 0 |
| 주택 담보 대출 기간 | 숫자 입력 | 년 / 월 토글 | 년 |
| 주택 담보 대출 금리 | 소수점 입력 | — | 0% |
| 주택 담보 대출 상환방법 | 토글 | 원리금균등 / 원금균등 / 만기일시 | 원리금균등 |
| 보유대출 연이자 상환액 | 금액 입력 | — | 0 |
| 보유대출 연원금 상환액 | 금액 입력 | — | 0 |
| 지역 구분 | 토글 | 수도권·규제지역 / 지방 / 기타(전국) | 기타 |
| 금리 유형 (스트레스 DSR) | 토글 | 변동형 / 혼합형 / 주기형 / 순수고정형 | 변동형 |

**스트레스 DSR 가산금리 — 정책변수 JSON (`dsr_stress_rules.json`)**

```json
{
  "last_updated": "2026-04-05",
  "기본_스트레스금리": {
    "수도권_규제지역": 3.0,
    "지방": 0.75,
    "기타_전국": 1.50
  },
  "대출유형별_적용비율": {
    "변동형": 100,
    "혼합형": 80,
    "주기형": 40,
    "순수고정형": 0
  },
  "DSR_한도": {
    "제1금융권": 40,
    "제2금융권": 50
  },
  "지방_유예기간": "2026-06-30"
}
```

> ⚠️ 스트레스 금리는 **반기별(6월/12월)** 금융위원회가 공시합니다. 정책변수로 관리하며 직원이 설정 화면에서 수정 가능합니다.

**계산 공식:**

```
[기본 DSR]
신규대출 연원리금 = 대출 계산기 공식으로 산출
DSR = (신규대출 연원리금 + 보유대출 연이자 + 보유대출 연원금) / 연소득 × 100

[스트레스 DSR]
기본_스트레스금리 = 지역별 lookup (수도권 3.0% / 지방 0.75% / 기타 1.5%)
적용비율 = 금리유형별 lookup (변동 100% / 혼합 80% / 주기 40% / 순수고정 0%)
실적용_가산금리 = 기본_스트레스금리 × 적용비율 / 100
심사금리 = 대출금리 + 실적용_가산금리

스트레스 연원리금 = 심사금리로 재계산
스트레스 DSR = (스트레스 연원리금 + 보유대출 연이자 + 보유대출 연원금) / 연소득 × 100
```

**결과 표시:**
- 신규대출 연원리금 상환액
- 기본 **DSR** (강조)
- 스트레스 DSR:
  - 지역 + 금리유형 (예: "수도권·규제지역, 변동형")
  - 기본 스트레스금리 + 적용비율 → 실적용 가산금리 (예: "3.0% × 100% = +3.0%p")
  - 심사금리 (예: "대출금리 3.5% + 가산 3.0% = 6.5%")
  - **스트레스 DSR** (강조, 별도 색상)
- ⚠️ 예상 금액 안내 문구
- ⚠️ 규정 기준일 표시 + 미갱신 경고

### 3.8 중도상환수수료 계산기

**설명:** 대출을 약정 기일보다 일찍 상환할 경우 발생하는 수수료 계산

**입력 필드:**

| 필드명 | 타입 | 옵션 | 기본값 |
|--------|------|------|--------|
| 상환금액 | 금액 입력 | 빠른 금액 버튼 | 0 |
| 수수료율 | 소수점 입력 | 정책변수 기본값 로드, 직원 수정 가능 | 0.58% (2025.1 KB 기준) |
| 대출일자 | 날짜 선택 | 캘린더 팝업 | 오늘 |
| 상환일자 | 날짜 선택 | 캘린더 팝업 | 오늘 |
| 수수료 부과기간 | 숫자 입력 | +1년 / +3년 / +5년 버튼, 직원 수정 가능 | 정책변수 기본값 |

**계산 공식 (Sliding 방식):**

```
수수료 부과기간일수 = 수수료 부과기간 × 365
수수료 부과 종료일 = 대출일자 + 수수료 부과기간일수
잔존일수 = 수수료 부과 종료일 - 상환일자

if 잔존일수 ≤ 0:
    중도상환수수료 = 0  (부과기간 경과)
else:
    중도상환수수료 = 상환금액 × ��수���율 × (잔존일수 / 수수료 부과기간일수)
```

> 📌 분모는 대출 만기가 아니라 **수수료 부과기간**(통상 3년=1,095일)입니다.
> 예: 3년 부과기간 중 1년 후 상환 → 수수료 = 상환금액 × 수수료율 × (730/1095)

**결과 표시:**
- 상환금액
- 수수료율
- 잔존일수 / 수수료 부과기간 (일수)
- **예상 중도상환수수료** (강조, 0원이면 "부과기간 경과로 수수료 없음" 표시)

### 3.9 환전/송금 계산기

**설명:** 외화 환전 및 해외 송금 시 원화/외화 변환 계산

**입력 필드:**

| 필드명 | 타입 | 옵션 | 기본값 |
|--------|------|------|--------|
| 거래 종류 | 토글 | 환전(매도) / 환전(매입) / 송금(보낼 때) / 송금(받을 때) | 환전(매도) |
| 통화 | 드롭다운 | USD, JPY, EUR, CNY, GBP 등 | USD |
| 외화 금액 | 숫자 입력 | — | 0 |
| 적용 환율 | 숫자 입력 (소수점) | — | 직원 수동 입력 |
| 우대율 | 숫자 입력 | 0~100% | 0% |
| 송금 수수료 | 금액 입력 | (송금 시만 표시) | 0 |

**환율 설정 — 외부 JSON 설정 파일 (`exchange_rates.json`)**

```json
{
  "last_updated": "2026-04-05 09:00",
  "rates": {
    "USD": { "매매기준율": 1380.00, "현찰_매도": 1404.42, "현찰_매입": 1355.58, "송금_매도": 1393.80, "송금_매입": 1366.20 },
    "JPY": { "매매기준율": 9.20, "현찰_매도": 9.43, "현찰_매입": 8.97, "송금_매도": 9.29, "송금_매입": 9.11 },
    "EUR": { "매매기준율": 1510.00, "현찰_매도": 1540.20, "현찰_매입": 1479.80, "송금_매도": 1525.10, "송금_매입": 1494.90 }
  }
}
```

> 📌 1차 버전: 직원이 설정 화면에서 수동 입력. 추후 인트라넷 기준환율 연동 탐색.

**계산 공식:**

```
스프레드 = (적용환율 - 매매기준율)
우대 적용 환율 = 매매기준율 + 스프레드 × (1 - 우대율/100)

[환전 매도 — 고객이 외화 구매]
원화 필요 금액 = 외화 금액 × 우대 적용 환율

[환전 매입 — 고객이 외화 판매]
원화 수령 금액 = 외화 금액 × 우대 적용 환율

[송금 보낼 때]
원화 필요 금액 = 외화 금액 × 우대 적용 환율 + 송금 수수료

[송금 받을 때]
원화 수령 금액 = 외화 금액 × 우대 적용 환율 - 송금 수수료
```

**결과 표시:**
- 적용 환율 (우대 반영)
- 외화 금액
- 송금 수수료 (해당 시)
- **원화 환산 금액** (강조)

| 이자소득 과세율 (일반15.4%/비과세/세금우대1.4%) | `tax_rates.json` | 현행 세법 | 세법 개정 시 (가능성 낮음) |
| 환율 | `exchange_rates.json` | 수동 입력 | **수시** |
| 중도상환 수수료율 기본값 | `prepayment_rules.json` | 0.58% (2025.1 KB 기준, 행내 확인 필요) | 상품 변경 시 |
| 중도상환 수수료 부과기간 기본값 | `prepayment_rules.json` | 통상 3년 (상품별 상이) | 상품 변경 시 |

### 5.3 설정 파일 구조

```
config/
├── ltv_rules.json          # LTV 비율 정책변수
├── dti_rules.json          # DTI 관련 정책변수
├── dsr_stress_rules.json   # 스트레스 DSR 가산금리 정책변수
├── tax_rates.json          # 이자소득 과세율 정책변수
├── exchange_rates.json     # 환율 (직원 수동 관리)
├── prepayment_rules.json   # 중도상환 수수료 정책변수
└── app_settings.json       # 앱 설정 (단축키, 창 크기, 테마 등)
```

> 직원이 설정 화면에서 수정 → JSON 자동 저장 → 앱 재시작 없이 즉시 반영
